01 — The Big Data Challenge

86.2 million rows.
Pandas would crash.

We start with the HEAPO dataset — nearly 1,400 households streaming a power reading every 15 minutes. That is roughly 86.2 million rows of raw consumption data. Loaded into a single-machine tool, the kernel runs out of memory and dies before the first chart renders.

So the system is built on Apache Spark from the ground up. Spark acts as a manager: it shards the dataset into small partitions and farms them out to a team of workers that process in parallel. This is horizontal scalability — we add workers, not bigger machines, and the pipeline keeps up.

03 — Defending against attacks

Assume the worst.
Filter the rest.

A real grid will be probed, hacked, and degraded. Multi-Krum keeps the federated model honest even when 20% of nodes lie.

Multi-Krum filter · simulation

Geometric distance reveals the liars.

For every round, each agent's update is scored by its distance to its closest neighbors. Updates that drift too far from the honest cluster are rejected before they reach the global model.

05 — Cheat sheet

Six terms,
plain English.

If you skipped the jargon, start here. Everything the pipeline depends on, defined without hand-waving.

01 · Scaling

Horizontal scaling

Add more machines, not bigger ones. A 16-core laptop hits a wall at 86 million rows; a Spark cluster shrugs.

02 · Joins

Broadcast join

Ship the small table (weather, ~24 rows/day) to every worker once. Skip the network shuffle. Hourly weather meets quarter-hour power readings without the cluster groaning.

03 · Sampling

Stratified sampling

Sample proportionally from each cluster instead of uniformly at random. Heavy users and apartments both make the cut — agents learn the whole population, not the loudest slice.

04 · Threat

Byzantine attack

A compromised agent that looks cooperative but ships poisoned model updates. In federated learning the server can't see the data, so it can't tell honest from hostile by inspection.

05 · Defense

Multi-Krum

Each round, score every update by its distance to its nearest neighbours. Keep the closest n − f; throw out the outliers. Survives up to f traitors per round.

06 · Metric

Recall

Of the real attacks, how many did we catch? In a defensive system, recall matters more than accuracy — missing a Byzantine is worse than flagging an honest worker by mistake.
